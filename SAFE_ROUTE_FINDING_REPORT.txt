OVERVIEW
================================================================================

Your application implements an intelligent alternative route analysis system 
that finds the safest maritime path by comparing parallel route options based 
on real-time weather and marine conditions.


================================================================================
1. CORE FUNCTIONALITY
================================================================================

----- Initial Route Scan -----
• User clicks destination → system samples route every 15 km
• Calculates ETA for each point based on boat speed (default 10 km/h)
• Fetches time-aligned weather forecasts for each point's arrival time
• Generates hazard classification for each segment

----- Weather Data Integration -----
• Atmospheric: Wind speed, gusts, precipitation, weather codes, visibility
• Marine: Wave height/direction, ocean current velocity/direction
• Risk Calculation: Combines wind risk + wave risk + weather risk 
  → unified hazard level


================================================================================
2. HAZARD CLASSIFICATION SYSTEM
================================================================================

----- Risk Levels & Scoring -----
LOW      = 1 point   (Safe conditions)
MEDIUM   = 2 points  (Moderate risk)
HIGH     = 5 points  (High risk)
DANGER   = 10 points (Dangerous conditions)

----- Risk Thresholds -----
Wind Speeds:
  • 0-20 km/h   = LOW
  • 20-40 km/h  = MEDIUM
  • 40-60 km/h  = HIGH
  • >60 km/h    = DANGER

Wave Heights:
  • 0-1 meters  = LOW
  • 1-2 meters  = MEDIUM
  • 2-3 meters  = HIGH
  • >3 meters   = DANGER

Weather Conditions:
  • WMO weather codes mapped to risk levels
  • Includes: thunderstorms, heavy rain, fog, etc.


================================================================================
3. ALTERNATIVE ROUTE GENERATION
================================================================================

----- Smart Triggering -----
• Only generates alternatives if main route has ANY risk (MEDIUM/HIGH/DANGER)
• If all segments are LOW risk → no alternatives shown (route is perfectly safe)
• Avoids unnecessary API calls and comparisons
• Optimizes performance and user experience

----- Parallel Path Algorithm -----
Main Route:   Direct path from A to B
North Route:  Offset +0.08° latitude (~9 km north)
South Route:  Offset -0.08° latitude (~9 km south)

----- Key Design Features -----
• Fixed endpoints: Start and end points remain unchanged
• Only intermediate waypoints shift: Creates arc-like parallel paths
• Same distance covered: Routes maintain similar travel times
• Weather variation: Different paths encounter different weather conditions


================================================================================
4. ROUTE COMPARISON & SELECTION
================================================================================

----- Automatic Scoring -----
Each route receives a hazard score (sum of all segment scores):

  Score = (LOW_count × 1) + (MEDIUM_count × 2) + 
          (HIGH_count × 5) + (DANGER_count × 10)

Lower score = safer route

----- Selection Logic -----
1. System sorts routes by score (lowest = safest)
2. Auto-selects safest route
3. User can manually switch between routes
4. Original route shown as dashed red line when alternative is active

----- Route Comparison Criteria -----
• Total hazard score
• Percentage of safe segments
• Count of each risk level
• Overall status (Safe/Moderate/High Risk/Dangerous)


================================================================================
5. VISUAL FEEDBACK SYSTEM
================================================================================

----- Map Visualization -----
Active Route: Thick colored segments (6px width)
  • Green  = LOW risk
  • Yellow = MEDIUM risk
  • Orange = HIGH risk
  • Red    = DANGER

Original Route: Dashed red outline (3px, 40% opacity)
  • Shows when viewing alternative
  • Allows visual comparison

----- Route Comparison UI -----
• Gradient banner showing safest route recommendation
• 3-column score comparison with visual hierarchy
• Individual route cards with:
  - Overall status badge
  - Percentage of safe segments
  - Hazard score
  - Breakdown of risk levels (LOW/MEDIUM/HIGH/DANGER counts)
  - Click to switch routes instantly

----- Warning System -----
• "No Safe Route Found" alert if all routes >40% high-risk segments
• Dynamic risk labels based on actual percentages:
  - "High Risk" (>20% HIGH/DANGER)
  - "Moderate Risk" (>30% MEDIUM+HIGH+DANGER)
  - "Elevated Risk" (default)


================================================================================
6. TECHNICAL IMPLEMENTATION
================================================================================

----- Distance Calculation -----
• Haversine formula for accurate maritime distances
• Output in Nautical Miles (1 NM = 1.852 km)
• Considers Earth's curvature for precision

----- Time-Aligned Forecasts -----
• Calculates arrival time at each point based on:
  - Cumulative distance traveled
  - Boat speed
  - Departure time
• Matches to nearest hourly forecast
• Ensures weather data corresponds to conditions when vessel arrives

----- Route Sampling -----
Interval:      15 km between sample points
Max Points:    100 points per route
Max Distance:  ~1500 km (100 points × 15 km)
Sample Method: Linear interpolation between start and end

----- Parallel Processing -----
• Fetches weather data for all points simultaneously using Promise.all()
• Analyzes 3 routes in sequence (main → north → south)
• Efficient API usage with Open-Meteo (free, no API key required)

----- State Management -----
Key State Variables:
  • routeWeatherPoints: Current active route's weather data
  • alternativeRoutes: Array of all analyzed routes
  • selectedRouteType: 'main' | 'north' | 'south'
  • originalMainRoute: Stores main route for comparison visualization
  • analyzingRoutes: Loading state during analysis

8. CURRENT WORKFLOW
================================================================================

Step-by-Step Process:

1. User clicks destination on map
   ↓
2. System scans main route at 15 km intervals
   • Calculates distance and ETA for each point
   • Fetches atmospheric + marine data
   • Generates hazard classifications
   ↓
3. Checks if route has any risks (MEDIUM/HIGH/DANGER)
   ↓
4a. NO RISKS FOUND:
    • Display single safe route
    • Show "perfectly safe" message
    • No alternatives needed
   
4b. RISKS DETECTED:
    • Generate 3 alternative routes (main/north/south)
    • Shift parallel routes by ±0.08° latitude (~9 km)
    • Keep start/end points fixed
   ↓
5. Compare routes by hazard score
   • Calculate total score for each route
   • Sort by score (ascending)
   ↓
6. Auto-select safest route
   • Set as active route
   • Display on map with colored segments
   ↓
7. Display comparison UI
   • Show banner with recommendation
   • Display 3 route cards with scores
   • Highlight safest option
   ↓
8. User can switch routes interactively
   • Click any route card
   • Map updates instantly
   • Original route shows as dashed red line

11. API INTEGRATION
================================================================================

----- Open-Meteo Weather API -----
Endpoint: https://api.open-meteo.com/v1/forecast
Data Retrieved:
  • temperature_2m
  • windspeed_10m
  • windgusts_10m
  • precipitation
  • weathercode (WMO standard codes)
  • visibility

----- Open-Meteo Marine API -----
Endpoint: https://marine-api.open-meteo.com/v1/marine
Data Retrieved:
  • wave_height
  • wave_direction
  • ocean_current_velocity
  • ocean_current_direction
  • wave_height_max (daily)

Advantages:
  • Free to use, no API key required
  • Hourly forecasts up to 7 days
  • Global coverage
  • High reliability
